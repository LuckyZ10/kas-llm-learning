# ML教材写作方法论 (Writing ML Textbook Methodology)

> 从《机器学习与深度学习：从小学生到大师》写作过程中提炼的标准方法论
> 目标：打造世界最伟大的机器学习教材

---

## 🎯 核心原则

### 1. 质量第一，慢工出细活
```
不赶进度 → 每章经得起推敲 → 质量下滑立即回炉
```

### 2. 边写边复盘，持续打磨
```
写新章时回顾前两章 → 每5章系统检查 → 发现不一致立即更新
```

### 3. 费曼学习法
```
抽象概念 → 生活化比喻 → 从具体到抽象 → 用人话解释
```

### 4. 数学推导完整
```
每一步解释"为什么" → 不跳步 → 数值示例验证
```

### 5. 手写代码完整可运行
```
NumPy手写开始 → 框架实现 → 逐步调试方法
```

### 6. 章节灵活扩展，不自我设限
```
有歧义不纠结 → 扩展章节容纳 → 内容自然贯通
```

**重要原则**：当对某个章节的内容范围有歧义时，不要强行压缩或取舍。宁可扩展章节数量（如从60章扩展到65章、70章），也要确保每个知识点都有足够的空间讲清楚。教材的完整性比章节数的整齐更重要。

## 📋 标准写作流程

### Phase 1: 深度研究 (2-3小时)
- [ ] 搜索8-12篇核心论文（近3年顶会：NeurIPS/ICML/ICLR/CVPR）
- [ ] 阅读经典文献，理解核心思想
- [ ] 绘制技术演进时间线
- [ ] 标注关键公式和算法

### Phase 2: 费曼法打磨 (1-2小时)
- [ ] 为核心概念设计3个生活化比喻
- [ ] 检验比喻：8岁小孩能听懂吗？
- [ ] 构建"直觉→形式化→应用"的叙述链条
- [ ] 参考比喻库，避免重复

### Phase 3: 数学推导 (2-3小时)
- [ ] 列出所有需要推导的公式
- [ ] 从零开始，不假设前置知识
- [ ] 每一步标注"为什么"
- [ ] 提供数值示例验证
- [ ] 绘制推导流程图

### Phase 4: 代码实现 (3-4小时)
- [ ] 设计代码架构（类/模块划分）
- [ ] 先写NumPy基础版本
- [ ] 再写PyTorch框架版本
- [ ] 添加可视化工具
- [ ] 编写可运行的示例脚本
- [ ] 测试所有代码

### Phase 5: 整合与润色 (1-2小时)
- [ ] 按模板组装章节
- [ ] 统一术语翻译
- [ ] 检查章节间引用
- [ ] 设计9道练习题（3基础+3进阶+3挑战）
- [ ] 整理APA格式参考文献

### Phase 6: 质量检查 (1小时)
- [ ] 运行质量检查清单
- [ ] 朗读测试（是否顺口）
- [ ] 同行评审（模拟读者视角）
- [ ] 记录经验教训到skill

## ✅ 质量检查清单

### 内容质量
- [ ] 费曼比喻是否生动？（8岁小孩能懂吗？）
- [ ] 数学推导是否完整？（每一步有解释吗？）
- [ ] 代码能否直接运行？（复制粘贴能执行吗？）
- [ ] 章节间引用是否连贯？（与前面章节有衔接吗？）
- [ ] 术语翻译是否统一？（全文一致吗？）

### 格式规范
- [ ] 标题层级正确（# ## ###）
- [ ] 代码块标注语言
- [ ] 公式使用LaTeX格式
- [ ] 图表有标题和说明
- [ ] 参考文献APA格式

### 教育价值
- [ ] 有明确的
学习目标
- [ ] 有逐步引导的思考过程
- [ ] 有常见错误和避坑指南
- [ ] 有扩展阅读建议
