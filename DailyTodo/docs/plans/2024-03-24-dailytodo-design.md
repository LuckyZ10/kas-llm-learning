# DailyTodo UI 设计文档

**创建日期**: 2024-03-24  
**技术栈**: Tauri + React + TypeScript + TailwindCSS  
**设计风格**: 现代简约风  

---

## 1. 概述

DailyTodo 是一款桌面端的日常任务管理工具，结合了可爱的"冰箱贴"视觉元素和强大的AI助手功能。

### 1.1 核心概念

- **日期导航**: 顶部两行可滑动选择器（月份 + 日期）
- **三栏布局**: 左栏（冰箱贴板 + AI对话）+ 右栏（Daily记录）
- **⭐ 冰箱贴板（核心亮点）**: 
  - 每个任务拥有一个专属的文件展示区
  - **自由摆放**: 文件可以拖拽到任意位置，像真实冰箱贴一样随意放置
  - **标签系统**: 支持文字标签 + 6种颜色标签分类
  - 支持拖拽文件，点击即可打开
- **AI助手**: 具备代码级操作能力，可创建、编辑、编排文件

### 1.2 目标用户

- 需要视觉化任务管理的个人用户
- 希望通过AI辅助完成编程或文件整理工作的开发者
- 喜欢可爱、直观界面的效率工具爱好者

### 2.3 方案选型理由

选择 **Tauri** 而非 Electron 或纯 Web 的原因：

1. **文件系统**: Tauri 原生支持本地文件操作，拖拽体验流畅
2. **性能**: 比 Electron 更轻量（安装包 < 10MB）
3. **安全性**: Rust 后端更安全，适合处理本地文件
4. **未来扩展**: 可轻松打包为移动端应用

### 3.2 组件分解

#### DateSelector（日期选择器）

**结构**: 顶部两行独立滑动区域
- **MonthRow**: 水平滑动，显示12个月份，当前选中高亮
- **DayRow**: 水平滑动，显示选中月份的日期，支持跨月滑动
- **交互**: 点击月份/日期切换，右栏自动更新对应日期的记录

**技术要求**:
- 使用 CSS scroll-snap 实现流畅滑动
- 支持触摸和鼠标拖拽
- 当前选中项居中显示，高亮主题色

#### LeftPanel（左侧面板）

**上半部分 - FridgeBoard（冰箱贴板）⭐ 核心亮点功能**:

冰箱贴板是 DailyTodo 的灵魂功能，模拟真实冰箱的使用体验：

**🎯 自由摆放系统**:
- 文件可以**随意拖拽到板上任意位置**（x,y 绝对坐标）
- 每个文件贴可以**叠放**（z-index 层级）
- 支持**旋转**（-8° ~ 8°），营造真实随意感
- 拖拽文件进入时自动计算一个"不错"的初始位置
- 已放置的文件可以继续拖动调整位置

**🏷️ 标签系统**:
- **文字标签**: 给文件贴添加说明文字（如"重要"、"参考"、"待处理"）
- **颜色标签**: 6种预设颜色快速分类：
  - 🔴 红色 = 紧急
  - 🔵 蓝色 = 参考
  - 🟢 绿色 = 已完成
  - 🟡 黄色 = 待处理
  - 🟣 紫色 = 想法
  - 🟠 橙色 = 进行中
- 标签显示在文件贴的角落或边缘

**📎 文件管理**:
- 点击右栏任务时，该任务对应的文件贴在此展示
- 文件以"可爱冰箱贴"样式展示：
  - 轻微旋转 + 投影效果（看起来像真的贴在冰箱上）
  - 圆角边框 + 不同文件类型显示不同图标/预览
  - 底部小磁贴效果（看起来像磁铁吸住）
- 支持拖拽新文件进来（复制到任务专属目录）
- 点击文件可打开预览或系统默认程序
- 右键菜单：编辑标签、调整大小、删除

**下半部分 - AIChat（AI对话）**:
- 固定高度对话框（可折叠）
- 上下文感知：能读取当前选中任务的描述
- 支持询问：
  - "我今天有什么任务"
  - "帮我总结一下"
  - "创建一个Python脚本处理这些文件"
- 消息气泡式布局，简洁清爽

#### RightPanel（右侧面板）

**DailyEditor（Daily记录编辑器）**:
- 富文本编辑，支持 Markdown 风格
- 任务列表结构：
  - 复选框（完成状态）
  - 任务标题
  - 可展开的描述区域
- 点击任务 → 左栏上半部分显示该任务的冰箱贴
- 支持子任务（嵌套）
- 底部有快速添加任务的输入框

## 4. 数据模型与数据流

### 4.1 核心数据模型

```typescript
// 任务数据结构
interface Task {
  id: string;              // 唯一标识
  title: string;           // 任务标题
  description?: string;    // 详细描述（富文本）
  completed: boolean;      // 完成状态
  createdAt: Date;
  updatedAt: Date;
  fridgePath: string;      // 该任务的冰箱贴目录路径
}

// Daily记录（按日期存储）
interface DailyRecord {
  date: string;            // YYYY-MM-DD
  tasks: Task[];           // 该日所有任务
  aiContext: string;       // 对话上下文摘要
  lastModified: Date;
}

// 文件贴（冰箱贴）- 核心亮点功能
interface FridgeItem {
  id: string;
  fileName: string;
  filePath: string;        // Tauri FS中的存储路径
  mimeType: string;
  size: number;
  previewType: 'image' | 'document' | 'other';
  createdAt: Date;
  
  // 🎯 核心功能1: 自由摆放
  position: {
    x: number;            // 绝对X坐标（px）
    y: number;            // 绝对Y坐标（px）
    rotation: number;     // 旋转角度 (-8° ~ 8°)
    zIndex: number;      // 层级（可叠放）
  };
  
  // 🏷️ 核心功能2: 标签系统
  tags: {
    text?: string;        // 文字标签（如"重要"、"参考"、"待处理"）
    color: 'red' | 'blue' | 'green' | 'yellow' | 'purple' | 'orange' | 'none';
                          // 颜色标签分类（红色=紧急、蓝色=参考、绿色=已完成等）
  };
  
  // 视觉样式
  visual: {
    magnetColor: string;  // 磁贴颜色（根据tags.color自动设置）
    shadow: boolean;      // 是否显示投影（可开关）
    scale: number;       // 缩放比例（0.5 ~ 1.5）
  };
}

// 预设标签颜色定义
```

## 7. 错误处理与边界情况

### 7.1 错误分类与处理

| 错误类型 | 场景 | 处理方式 |
|---------|------|---------|
| **文件系统错误** | 拖拽文件失败、存储空间不足 | 弹窗提示 + 自动重试机制 |
| **AI服务错误** | API调用失败、网络中断 | 降级到本地模式（离线基本功能） |
| **数据同步错误** | IndexedDB操作失败 | 自动备份到本地JSON文件 |
| **权限错误** | Tauri文件访问被拒绝 | 引导用户开启权限 |
| **未知错误** | 运行时异常 | 错误日志 + 崩溃恢复 |

### 7.2 边界场景处理

**场景1**: 用户拖拽超大文件 (>100MB)
- 检测文件大小
- 提示: "文件过大，是否只保存链接？"
- 选项: [保存链接] [取消] [压缩后保存]

**场景2**: AI生成代码有语法错误
- 自动语法检查失败
- 尝试自动修复一次
- 失败则标记为"待修复"
- 在对话中告知用户并提供修复建议

**场景3**: 切换日期时当前任务未保存
- 自动保存草稿到 IndexedDB
- 下次打开该日期时恢复
- 显示提示: "已恢复上次未保存的内容"

## 9.5 冰箱贴板交互详解 ⭐

### 9.5.1 文件拖拽与摆放

**拖入文件**:
1. 用户从系统文件管理器拖拽文件到冰箱贴板
2. 拖拽过程中板子显示高亮边框和"释放以添加"提示
3. 释放后，文件自动复制到任务专属目录
4. 新文件贴出现在释放位置附近（自动避开已有文件）
5. 播放轻快的"贴上"音效（可选）

**移动文件**:
1. 长按或点击选中文件贴（显示选中边框）
2. 拖拽到任意位置
3. 释放后更新 position.x/y 坐标
4. 支持叠放（自动递增 z-index）

**双击旋转**:
- 双击文件贴可随机调整旋转角度
- 或按住 Alt 键拖拽来调整旋转

### 9.5.2 标签管理

**添加/编辑标签**:
1. 右键点击文件贴 → 选择"编辑标签"
2. 弹出小面板：
   - 文字输入框（最多10个字符）
   - 6色颜色选择器
   - 清除标签按钮
3. 实时预览标签效果
4. 确认后保存到 index.json

**标签显示**:
- 文字标签显示在文件贴右下角（小型胶囊样式）
- 颜色通过文件贴边框和磁贴颜色体现
- 多个文件同标签时，视觉上形成"组"

**标签筛选**:
- 板子顶部有标签过滤器（横向小图标）
- 点击颜色筛选显示该颜色的所有文件
- 支持多选（显示红色 AND 蓝色文件）

### 9.5.3 快捷操作

**右键菜单**:
- 编辑标签
- 调整大小（0.5x, 0.75x, 1x, 1.25x, 1.5x）
- 置于顶层/底层
- 删除（确认弹窗）
- 打开文件所在位置

**快捷键**:
- `Delete` - 删除选中的文件贴
- `Ctrl+Z` - 撤销上次移动/旋转
- `Space` - 预览文件（图片放大，文档快速查看）

### 9.5.4 视觉细节

**真实感营造**:
- 轻微阴影（看起来真的贴在冰箱上）
- 不同文件类型有不同"厚度"（图片较薄，PDF较厚）
- 磁贴部分（顶部）颜色略深，像磁铁
- 鼠标悬停时轻微浮起（z轴位移）

**动画效果**:
- 拖入新文件：从0缩放到1，带弹跳效果
- 移动文件：平滑跟随鼠标，释放时轻微晃动
- 删除文件：缩小消失 + 粒子效果（可选）
- 切换任务：旧文件飞走，新文件飞入

### 11.2 设计决策记录

1. **为什么选择Tauri而非Electron?**  
   - 更小的安装包（<10MB vs >100MB）
   - 更低的内存占用
   - Rust后端更安全

2. **为什么冰箱贴要复制文件而不是链接?**  
   - 原文件可能被移动或删除
   - 每个任务有独立的文件空间，便于整理
   - 支持离线访问

3. **为什么AI需要分级权限?**  
   - 避免误操作（自动删除文件很危险）
   - 给用户掌控感
   - 符合用户期望（简单操作自动化，危险操作需确认）

4. **为什么冰箱贴板要支持自由摆放而不是列表?**  
   - 模拟真实冰箱的使用体验，更直观有趣
   - 自由摆放更符合人类的空间记忆习惯
   - 视觉化的信息组织方式，一眼就能看出文件关系
   - 这是产品核心差异化亮点，区别于传统文件管理器

5. **为什么标签系统要同时支持文字和颜色?**  
   - 颜色标签：快速视觉分类（一眼识别紧急/完成状态）
   - 文字标签：提供具体语境（如"客户A的反馈"）
   - 两者结合：既有直观性又有精确性
   - 6色预设：足够覆盖常见分类场景，避免选择困难
