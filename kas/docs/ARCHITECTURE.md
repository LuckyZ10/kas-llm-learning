# KAS (Klaw Agent Studio) 架构文档

> 版本: v1.4
> 最后更新: 2026-03-23

---

## 🎯 核心理念

**KAS = OpenClaw 特种部队指挥系统**

```
代码吞食 → 能力提取 → Agent 进化 → 团队协作
```

**与传统平台的区别**:
| 特性 | Dify/Coze | KAS |
|------|-----------|-----|
| 能力来源 | 预设组件 | 从实战代码提取 |
| Agent 创建 | 拖拽组装 | 代码吞食 + 合体 |
| 多 Agent | 工作流包装 | 沙盒化隔离协作 |
| 目标用户 | 业务人员 | 专业开发者 |

## 🎯 设计原则

### 1. 简单优先
```
能用规则解决 → 不用 ML
能用 ML 解决 → 不用 DRL
```

### 2. CLI First
```
开发者最爱的交互方式
所有功能必须可通过 CLI 访问
Web UI 作为辅助
```

### 3. 代码即知识
```
从实战代码提取能力
而非预设模板
Agent 越用越强
```

### 4. 可组合
```
Agent 可以合体
能力可以流通
装备可以扩展
```

### 5. 沙盒化优先
```
没有沙盒的"Agent团队"只是工作流包装
必须先实现沙盒隔离
才能安全协作
```
